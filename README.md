Projeto para testes em simula um chat inteligente que com uma conversa natural com o paciente vai marcar consulta, desmarcar consulta, cadastrar paciente, ver agenda médica disponível, ver consultas marcadas.
O projeto vai ser guiado por fluxos mermaid, dividindo o SYSTEM_MESSAGE em varias partes para diminuir o contexto, o primeiro passo recuperar os dados do paciente, estando ele cadastrado deve recuperar pelo CPF, se não tiver no banco de dados deverá solicitar os dados para cadastro, após recuperar deverá mostrar o menu de opções com o que pode fazer, já descrito acima (marcar consulta, desmarcar consulta, ver agenda médica, ver consultas marcadas), passo do cadastro e consulta do paciente já é implicito para fazer qualquer outra opção tem que ter o id do paciente.
O sistema não rodará inteiramente no contexto da IA, exemplo: a IA conversará com o paciente recuperarndo o CPF, irá chamar uma tool e com o retorno irá armazenar em uma variável ou em um json para ser enviado para o browser ou api. detectando os dados do usuário já apresenta o menu que é uma nova chamada para outra interação com o LLM, não passando o historico anterior, mas apenas os dados do paciente e apresentando o menu, não sei a IA apresenta o menu ou se o python, e dependendo da escolha a IA chama outro subfluxo com sua SYSTEM_MESSAGE, as SYSTEM_MESSAGE estão em prompt.py
